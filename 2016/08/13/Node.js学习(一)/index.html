node简介

  1. Node.js不是js应用,而是js运行平台(JavaScript runtime)
  2. 构建在Chrome’s V8 JavaScript engine之上
  3. 每个函数都是异步的
  4. 使用npm作为包管理工具

  5. 作用:网站(express/koa),im(socket.io),api,http proxy,前端构建工具(grunt/bower),操作系统(NodeOS),跨平台打包工具(nw.js),命令行工具(cordova)

    https://github.com/sindresorhus/awesome-nodejs
    http://stackshare.io/nodejs/in-stacks

npm

是NodeJS包管理和分发工具,作用与Ruby的gem、Python的pypl或setuptools、PHP的pear和.Net的Nuget一样。

install

  1. 安装nvm(node版本管理)

    curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.31.4/install.sh | bash

  2. 安装node版本(node/npm)

    nvm install 4.4.7(版本号)
nvm alias default 4.2.0(切换默认版本)

  3. 安装nrm (切换node源的模块)

    npm install -g nrm

CommonJS规范

  1. require : 用来引入依赖

  2. export : 用来导出模块

    1. module.exprots

    2. exports.xxx

      如果你想你的模块是一个特定的类型就用module.exports。
      如果你想的模块是一个典型的“实例化对象”就用exports。

编写nodejs模块

  1. 在github.com上创建仓库
  2. 通过git clone https://github.com/jsntjinjin/demo_cli.git 下载
  3. cd demo_cli之后npm init,生成package.json文件

  4. 创建index.js

    #!/usr/bin/env node
console.log('hello node module')

  5. 在生成的package.json中添加:

    "preferGlobal": "true",
"bin": {
    "stuq": "stuq.js"
},

    preferGlobal确定你的这个命令是不是全局的，一定要设置为true，不然不放到path里，不能全局用的。
    bin是配置你的cli名称和具体哪个文件来执行这个的

  6. 通过git上传到github.com上

    git status
git add .
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
git commit -m "first commit"
npm login //登录到npm
npm publish . //发布到npm上
npm install -g "xxx"//下载module
