Android布局优化

发表于 2016-07-23   |   分类于 android   |     |   阅读次数

本篇文章主要学习了Android布局优化相关方面的知识.

  1. 优化渲染时间
    在Android中,系统在渲染的时候,会以1000ms渲染60帧的速率即16ms的时间来渲染界面,当我们每次渲染的时候,如果渲染时间总是小于16ms的话,我们的界面会非常流畅;如果我们对于渲染的界面逻辑非常复杂的话,大于16ms,系统将丢帧,系统将会等待下次的信号才会绘制,即16ms*2的时间显示同一帧,导致界面卡顿.
    在开发者选项中,提供了检测UI渲染时间的工具,GPU呈现模式分析中的在屏幕上显示为条形图.
    打开的时候界面上会出现很多的柱状条,蓝色代表绘制的呈现时间,红色代表OpenGL渲染呈现时间,黄色代表CPU等待GPU渲染的时间,中间的绿色横线代表16ms,我们需要将自己的UI渲染时间尽量控制在绿色横线下面.

  2. Overdraw
    当我们在布局中过度绘制的时候,会大量浪费CPU和GPU资源,
    在开发者选项中,提供了检测Overdraw的工具,调试GPU过度绘制.通过这个工具,尽量减少红色区域,增大蓝色区域.

  3. 优化布局层级
    系统对于View的测量布局绘制,都是通过遍历View数来操作的.如果一个View数过大的话,就会严重影响测量布局绘制的速度.

  4. 减少嵌套过多的无用的布局

    • 使用<include>标签重用Layout

    • 使用<ViewStub>实现View的延时加载
      ViewStub是一个轻量级的组件,不可见,并且大小为0.

      1. 在布局中使用,和正常组件一样,但是需要制定layout属性.
      2. 通过findViewById()找到ViewStub.

      3. 两种方式显示

        • mViewStub.setVisibility(View.VISIBLE);
        • View inflateView = mViewStub.inflate();

      4. 通过inflate()方法可以获得ViewStub的布局,通过这个布局我们可以对布局中的组件使用.

      5. 当ViewStub显示之后,ViewStub就不存在了,存在的只有inflate()之后的布局.所以不能调用两次inflate()方法.
