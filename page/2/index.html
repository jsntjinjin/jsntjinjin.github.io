Handler机制浅析

发表于 2016-05-14   |     |   阅读次数

消息机制概述

Android的消息机制主要指的是Handler的运行机制,及其Looper和MessageQueue的工作的状态.

Handler的主要工作内容是将任务切换到指定的线程中去执行.这主要是因为Android只能在主线程中访问UI,如果不在主线程中访问UI,程序将抛出异常.但是Android又不建议在主线程中执行耗时的操作,否则会产生ANR导致程序无法响应.

Android为什么不能在子线程中执行耗时操作呢? 因为Android的UI空间不是线程安全的,如果多线程并发访问,会导致UI错乱. 但是Android不加入锁机制的原因:

  1. 会导致UI访问的逻辑变的复杂
  2. 降级UI访问的效率

自定义带删除功能的EditText

发表于 2016-04-28   |     |   阅读次数

提出问题

如图所示,是一个我们现在应用中经常使用的一个控件,一个带删除功能的EditText,可能在你自己的app中所有的EditText控件都带删除功能,如果我们笨笨的使用线性或者相对布局来排列这三个控件,这样会出现两个问题:

  1. 代码过长,不易于查看,想想如果一个界面里面出现3个以上这种控件,xml布局中会出现很冗长的重复代码;

  2. 同样不易于维护,当产品要对整个app的EditText做出UI的修改的时候,你会发现很痛苦的在一个一个xml布局的修改,因为一个app要保证风格的统一,所有的对应的控件都需要修改;

    所以当我们发现一个app的很多界面有相同的控件时,我们就要留心将他们封装成一个自定义的控件,在使用的时候我们就复用这个控件就可以了.
